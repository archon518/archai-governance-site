What ARCHAI Is

Not a policy.
Not a framework document.
Running infrastructure.

ARCHAI is a deployed governance proxy — a live system that intercepts every AI request before it reaches the model, runs it through a five-mechanism governance chain, and produces a cryptographic record of what was authorized, by whom, and why.

It is not software you install and configure once. It is infrastructure that operates continuously, governing every request your AI systems make — with no exceptions and no bypasses.

The Governance Chain

Five mechanisms.
One unbroken sequence.

Every request passes through all five in order. No mechanism can be skipped. No action executes without completing the full chain.

01
IDENTARCH →
Identity Substrate
Establishes and verifies the cryptographic identity of every node making a request. Issues a time-bounded anchor that all subsequent mechanisms require. Without a valid anchor, the chain does not start.
02
INTENTUM →
Intent Binding
Captures the declared purpose of the request and binds it cryptographically to the verified identity. Actions that deviate from their declared intent are blocked. Unjustified actions do not proceed.
03
AGENTUM →
Execution Governance
Routes validated requests through registered, authorized pipelines only. Executes each step in sequence and halts the entire chain on failure. No action runs outside a pre-registered pipeline.
04
MNEMARCH →
Memory Governance
Writes an attributed, schema-validated memory record of every governed action — bound to the identity that authorized it, with a retention policy and decay schedule. Memory without authorization cannot be written.
05
ACCOUNTUM →
Accountability Ledger
Records every governance event to an immutable, hash-chained ledger. Each entry is cryptographically linked to the previous. Tampering with any entry breaks the chain and is detected instantly on verification.
5
Governance Mechanisms
Every request passes through all five in sequence. The chain cannot run out of order or skip a mechanism.
0
Ungoverned Requests
There are no exceptions. Every AI request your application makes produces a complete governance record.
∞
Tamper Evidence
The accountability ledger is hash-chained. Any modification to any historical entry is detected on the next verification.
